# marketing

Remotion 4 project that renders the marketing video for `@web-kits/audio`.

The trick: every sound you hear in the video is **the actual library** running.
A pre-render script (`scripts/bake-sounds.ts`) loads the patches from
`.web-kits/*.json`, calls the same `renderToBuffer()` the library exports, and
writes WAV files into `public/sounds/<patch>/<sound>.wav`. The Remotion
composition then mounts each WAV with `<Audio>` from `@remotion/media` at the
exact frame the on-screen interaction happens.

## Commands

```bash
pnpm --filter @web-kits/marketing studio          # interactive preview (Remotion Studio)
pnpm --filter @web-kits/marketing bake            # re-render WAVs from .web-kits/*.json
pnpm --filter @web-kits/marketing music           # generate the backing track (cached)
pnpm --filter @web-kits/marketing render          # 1920x1080 mp4 → out/launch-video.mp4
```

`bake` and `music` both run automatically before any `render` script.

## Source layout

```
src/
  index.ts             # Remotion entry — registerRoot()
  root.tsx             # <Composition id="Marketing"> wrapper
  marketing.tsx        # the composition tree (TransitionSeries of scenes)

  audio/
    backing-track.tsx  # ducked ambient pad
    cues.ts            # global cue schedule (folds per-scene CUES)

  config/
    timing.ts          # FPS, scene durations, transition length
    tokens.ts          # palette + spring presets

  generated/           # auto-written by scripts/, do not edit
    music.ts           # by scripts/generate-music.ts
    sounds.ts          # by scripts/bake-sounds.ts

  primitives/          # reusable interaction primitives
  scenes/              # one folder per scene
  types/               # ambient .d.ts shims
```

## Backing track

A soft ambient pad sits under the whole composition and ducks automatically
around every SFX cue. The track is generated by the ElevenLabs Music API and
cached to `public/music/backing.mp3`.

```bash
export ELEVENLABS_API_KEY=…
pnpm --filter @web-kits/marketing music
```

The script hashes the prompt + duration + model and skips the API call when
the cache is fresh, so re-renders are free. The committed `backing.mp3`
means teammates and CI can render without an API key. Edit the `PROMPT`
constant in `scripts/generate-music.ts` and re-run to regenerate.

Cue timings live in each scene's `CUES` export and are folded into a global
list by `src/audio/cues.ts`. The `<BackingTrack>` component in
`src/audio/backing-track.tsx` reads that list and computes a per-frame
`volume` envelope (6-frame attack, hold for the cue duration, 14-frame
release, ducks to 35% of the base level).

## Storyboard

Scenes mount in this order via `TransitionSeries` in `src/marketing.tsx`, with a
short crossfade between each:

| Scene | Sounds |
| --- | --- |
| ColdOpen | — |
| Introducing | — |
| Primitives | playful/click, playful/checkbox, playful/tab-switch, playful/toggle-on |
| Outro | playful/success |

Scene timings live in `src/config/timing.ts`. Animation primitives are in
`src/primitives/`. Each scene composes them and places `<Audio>` cues on the
exact frame an interaction "lands."

## Adding a sound to a scene

1. Author the patch in `.web-kits/<patch>.json` (or use an existing one).
2. `pnpm --filter @web-kits/marketing bake` to regenerate WAVs + `src/generated/sounds.ts`.
3. In a scene, import from `../../generated/sounds` and wrap an `<Audio>` in a
   `<Sequence from={hitFrame} layout="none">`.

## Why bake instead of synthesizing live?

Remotion renders frame-by-frame in headless Chrome. Web Audio works there, but
the audio track is encoded separately, so triggering one-shot sounds at exact
frames is much more reliable when each sound is a static asset with a known
duration. Baking also means the video is fully reproducible — same patch JSON
produces the same WAV bytes every render.
